## Usage Guidelines

### For Translators

**Keep Unchanged**:
- Brand names (GitHub, GitLab, Android, etc.)
- Standard acronyms (API, CI/CD, SSH, etc.)
- Git commands (`git commit`, `git push`, etc.)
- Code examples
- URLs and links

**Translate**:
- Explanatory text
- Descriptions
- User interface labels
- Error messages
- Help text
- Documentation prose

**Localize**:
- Date/time formats
- Number formats
- Currency
- Cultural references
- Examples (use local company names, etc.)

### For Technical Writers

**Consistency Rules**:
1. Use the same term throughout a document
2. Follow this glossary for standard terms
3. Define new terms when first introduced
4. Link to glossary for detailed definitions
5. Consider cultural context in examples

**Avoid**:
- Mixing synonyms (e.g., "repo" and "repository")
- Creating new terms unnecessarily
- Using jargon without explanation
- Ambiguous terminology
